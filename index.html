懂营销的前端程序员

发表于 2017-05-28

#Jup俊

##营销就是不流血的战争
也许你会好奇，一个程序员不好好敲代码搞业务逻辑，学什么营销？其实原来我也是这么想的，以前的我几乎只读程序有关的书籍，也许人生就是这么奇妙，工作中一次偶然的机会让我接触到了营销，让我看到了营销的璀璨魅力和让人惊叹的能量，我从此入坑，一发不可收拾，在写代码累了或者回家无聊时我看的都是营销的书。时间飞逝，一晃两年过去了，我觉的也应该试着去输出一些东西，特别是应该帮助有着理科本质一心向往技术的程序员兄妹学习营销，也许你学完成不了营销大师，但你的程序生涯绝对会混的风生水起。

##营销是你终身从事的事业，除非你是这两种人。

营销是每个人终生的事业，为什么这么说那？因为营销对我们普通人来说太重要了，他是我们突破社会竞争改变命运的战略性知识。

在我看来只有两种人不用学营销：

  • 富可敌国的富二代：比如国民老公王思聪，出生就含着金钥匙长大，一辈子就算什么都不干，也可以享尽人间欢乐。但王思聪显然不是一个不学无术的人，从他每次的宣传手段和上头条的频率就可以看出，他是一个营销高手，甚至说是一个营销大师。
  • 天才产品设计师：比如我们都知道的乔布斯，他设计的产品一出来就会让世界惊叹，有些人会排队几天抢首发，甚至一些疯狂的少年会割肾去买iphone。就算是这样的天才设计师，他能取得今天的成功也是有一支强大的营销团队在后边支撑的。
    ######如果你和我一样，是个普通呆板的程序员，没有华丽的出身和伟大的思想，那你一定要学营销。营销，营销，就是经营销售，销售是需要经营的，在营销人眼里天下没有不好的产品，只有不好的营销。当然无论你营销什么，都必须符合国家的标准原则，不能和国家标准向冲突，做人做事也是一样。

这个教程或者说是文章，个人觉的不仅仅局限于程序员来学习，如果你遇到了下面这些问题，就有必要学习一下营销知识：

  • 工作拼，却不被提拔。
  • 感情真，却没有对象。
  • 产品好，却卖不出去。
  • 组织棒，却没有名气。
    ######这些问题其实归纳起来就是一个问题：自身优秀和让别人觉的你优秀是两回事。我作了10年的程序员，每天在显示器前和计算机打着交到，你有没有想过，可能我们在销售，在产品和在老板眼里就是跟计算机一样会编程的机器。我想大家都不想成为这样的人，那开始跟着我一起学习营销吧。

##战争是流血的经济，营销是不流血的战争

人类自古以来就进行着残酷的财富争夺，大致国家，小致个人，都在进行着财富的争夺，这种争夺一刻也没有停止过。

财富争夺战有两种基本形式：

  • 战争：也叫做硬战争，我们一般说的战争说的都是流血的战争，它的组织形式是军队，杀戮是它的基本手段，争夺的是国家的经济资源。
  • 营销：也叫做软战争，营销是不流血的战争，它的组织一般是企业，它的基本手段是营销竞争，它争夺的是市场资源。
    ###如果你读过《人类简史》这本书，你会知道人类的发展可以分成三个阶段：
  1. 狩猎时代：人类主要靠狩猎和采摘来生存，这时候争夺的是野兽和野果，身体强壮占有很大优势工具是弓和剑，冷兵器。
  1. 农耕时代：人类开始大面积种植，驯服了可以养殖的动物，这时候争夺的是土地，工具变的更高级，后期出现了大型暴力工具和农耕机械。
  1. 现代社会时代：出现了实际意义上的共同体，国家。这时候人们的争夺更加残酷激烈，争夺的是领土和财富。这时候人们有能力组织几十万人为了争夺利益进行杀戮。现代聪明的人已经把财富争夺升级成了品牌的战争。
    ######硬战争是野蛮的，血淋淋的，让我们不寒而栗，人类唾弃的。比如二次大战，日本从我们中国掠夺走的真金白银达到500多亿，其他资源更是多的不计其数，烧杀抢掠，人人恨之入骨。在这个世界上，战争一刻都没有停止过：2001年阿富汗战争，2003年伊拉克战争，2008年俄格战争（俄罗斯和格鲁吉亚），2011年利比亚军事干预。

######与此同时这个世界上还进行着一场规模更大范围更广的永不终结的软战争，这就是营销。我们想象一下，我们喝着可口可乐这种黑褐色的液体的时候，你不会想象出我们中国的财富正源源不断的流向美国，当日本的汽车和三星的手机在中国销售时，几千亿几千亿的财富流向了日本和韩国。当这种财富流出时，我们并没有唾弃和烦感，而是有着敬畏和崇拜，我们说他们是令人尊敬的公司。

在硬战争和软战争中同样是日本运汽车到中国这件事,但是有着三个根本的不同：

  1. 以前日本是用战争的手段，而现在是用营销的手段。
  1. 以前日本带走的是头颅和鲜血，而现在带走的是中国人的大量财富。
  1. 以前我们仇恨日本，而现在我们接受或者说感激这些公司。
    #######通过上边的解说你可以明白，我们每个人都处在软战争中，无法避免只能接受，而我们程序员利用互联网和计算机制作着最先进的软战争武器，电子商务，互联网，人工智能，共享经济，自动驾驶，所以对于程序员来说，学会营销是你改变命运和走出困境的必会知识。我们反观现在的互联网大佬，它们是技术出身，但他们更是营销大师：
  • 程序员李彦宏学会了营销，成立了百度，走向了人生巅峰。
  • 程序员雷军学会了营销，成立小米公司，成为中国最伟大的手机公司。
  • 程序员马化腾学会了营销，成立了腾讯公司，成为世界上赫赫有名的网络社交公司。
  • 程序员周鸿祎学会了营销，成立了360公司，人民最喜爱的互联网安全公司。
    #######我们一定要打好软战争这场仗，往大了说它可以帮助国家在财富的争夺上有更大的优势，往小了说它能让我们这些程序员迎娶白富美，走向人生巅峰。记住这句话：营销是每个人终生的事业。

##日韩流入侵中国的真相 程序员全面沦陷
这章说的主要是日韩流对中国的入侵，我也知道这个时候讲日韩流会让大家不开心，毕竟萨德部署的风波还没有结束，日本也是对我们狼子野心。但是我们必须觉醒起来，看清楚我们国家和日韩营销的差距，并迎头赶上，构建我们国家的软实力，最终超越他们，利用软战争，提高我们的国际地位和竞争力。

##《来自星星的你》一部韩国宣传片

几个月前我们公司的美女甚至是少妇都疯狂迷恋上了都教授。都教授只要在电视里一出现，我们公司里外向的小婧就会手舞足蹈的向大家嚷起来：“快看，我老公又上电视了”。我这人很少看电视剧的，但当周围的人都在大谈特谈都教授时，我觉的自己out了，所以请教了小婧，她介绍我看了《来自星星的你》这个韩国电视剧。后来我才知道我们公司上到老总下到保洁阿姨，都在看这部电视剧。

这部电视剧真的让我很震撼，因为在我眼里它并不是一步电视剧，而是一部长达几十集的韩国广告宣传片。这部片子里无论好人坏人，风景和穿衣打扮每个画面都美到极致，所以这部电视剧展示的是一个广告，主题就是美到极致的韩国。

##那我们仔细研究一下这些营销元素？

  • 人美 ：即使在片中的反面人物李载京，都是潇洒英俊大长腿啊！但真实的韩国是这样吗？如果你去过韩国你就会发现，韩国没有那么多的帅哥美女，他们和我们相差无几。那为什么电视剧会把里边人物都塑造的那么美？因为韩国要推广他们的护肤品产业和整容产业，它让观众知道的就是，你可以不漂亮，但韩国会帮你整的很漂亮。就是这样潜移默化的营销，让中国每年上百万的女孩子蜂拥去韩国整容。
  • 景美 ：这部电视剧的每个镜头里的背景都非常美丽，就算是一个吵架的剧情，背景也会精心挑选美丽的景色，有美丽的树河流。就算两个人去吃一个路边摊，环境都是一尘不染，非常干净的。反观我们的电视剧大姨大妈吵架时一般选的环境就是街景小巷，不加任何修饰，甚至有垃圾堆在旁边；几个醉汉打架时，这小饭店的环境也是脏乱差，这都是没有注意细节，也没有任何美好的形象。宣传韩国的景色美丽，环境优雅是为了营销他们的旅游业。
  • 本土品牌 ： 这部电视剧的镜头经常会拍摄韩国的泡菜，三星手机，汽车上。这样的做法就是为了宣传韩国的知名品牌。韩国的汽车产业和中国的汽车产业几乎是同时起步，但是韩国铁矿资源匮乏，甚至自己不生产钢铁，但就是这样，韩国利用自己的营销手段，把自己的汽车产业打造成了世界著名，北京街上跑的大部分出租车都是韩国的现在汽车。

######韩国的一部电视剧都有这样的营销思想，是因为韩国把营销作为民族战略，韩国的前文化部部长说过三句很出名的话：

  1. 19世纪是武力征服世界的世纪，也就是二战前后，那时候的韩国落后了。
  1. 20世纪是品牌征服世界的世纪，因此韩国集国家的力量打造了一个又一个知名的品牌，像三星、LG和现代这样伟大的品牌。
  1. 21世纪是文化征服世界的世纪，韩国又集国家的力量打造了一部又一部好看的电视剧。
    ######你可能不知道，韩剧刚进入中国的时候，电视台播出不仅不用付钱，韩国还需要给我们的电视台钱，但是现在变了，每一部高质量的韩国电视剧都可以在中国赚得盆满钵满。这些电视剧不只在中国取得了很好的成绩，在全世界也有深远的影响。在奥运会期间，外国运动员用了韩国的技术作了针灸，美国人信服了韩国中医是世界上最好的，我想这些都跟电视剧的营销是分不开的。

如果你是一名程序员，请你观察一下身边的女性，他们用的化妆品、用的文具和服饰，你会发现身边很多人已经被韩流影响。一部电视剧就可以轻松的俘获你身边的女性，如果你还是单身，我想你是有必要学好营销的，毕竟关系到你另一半。

##日本的动漫和游戏产业

######营销的力量是无穷的，它甚至可以和枪炮媲美。日本的一位前首相就大力呼吁日本的企业要大力发展动漫和游戏产业，让中国的孩子们喜欢日本。当越来越多的中国孩子喜欢的日本的动漫和游戏时，他们就会少一些对中国的仇视。几十年过去了，日本的动漫和游戏已经被中国的孩子所接收，在中国出现了一大批二次元文化的崇拜者。

######日本的动漫和游戏产业也把所有的程序员沦陷了，我所认识的程序员就算不喜欢动漫，也会对游戏爱不释手，很多程序员拿了年终奖金后的愿望就是买个最新款的PS主机。就连最近很火的一款中国人开发的手游《阴阳师》里边也充满了日本动漫的元素。也许你会说，这款游戏就是给日本人玩的，但为什么在中国就火了起来。还是因为日本营销文化的影响已经深入中国青年内心。这就是日本营销的高明和可怕之处。

我是仇视日本的，我甚至有些抵抗日货的小民族情节。但是我们要惊醒自己，让自己在这场软战争中取得胜利。

##总结

中国现在很火的电视剧《琅琊榜》、《青云志》、《三生三世》，这些都是讲的玄幻过去的事情，这里我们的时代太远了，没有办法很好的营销我们的国家。韩国的电视剧都是现代的，这有利于营销宣传。好这章就说到这里，无论你是程序员或者其他职位，我都希望你能看明白文化营销的重要性。

1、AJAX基本组成（170204）

（1）、Ajax，异步请求，实现页面的局部刷新；
    客户端（Client）：前端，iOS，安卓；
    服务器（Server）：服务器集群；
            系统：Windows Server、linux（常用）；
            作用：提供数据，存储数据；
（2）、URL:
    基本概念:
            •   URL 的全称是 Uniform Resource Locator（统一资源定位符）
            •   通过 1 个 URL，能找到互联网上唯一的 1 个资源
            •   URL就是资源的地址、位置，互联网上的每个资源都有一个唯一的 URL
    URL格式:
            •   基本格式：协议头://主机地址/路径
                ◦   协议头：不同的协议，表示不同的资源查找及传输方式
                ◦   主机地址：存放资源主机的 IP 地址（域名）
                ◦   路径：资源在主机中的具体位置

HTTP协议

http，Hypertext Transfer Protocol，超文本传输协议；

请求

请求行：写明了请求方式（get/post）和请求路径（url）；
请求头：关键，包含了Content-type等信息，
            客户端环境描述、客户端请求的主机地址信息、
            客户端所能接收的数据类型，Accept:text/html；

请求体：请求方式为get时其请求体为空，post时请求体里存放了需要传输的数据，例如需要上传的数据；
    tips：post需要设置请求头，get不需要设置；

响应

响应行/状态行：写有状态码和http版本号；
            HTTP协议版本，状态码，状态码英文名称；

响应头：Content-type，记录了响应体中返回数据的文件格式；
响应体：服务器返回的数据，格式为JSON或XML；
    Content-Type:text/html;
                text/css;
                text/xml;
                text/html;
        tips：这些Content-type都统称称为MIME类型；

Wamp配置

w:windows,
a:apache,
m:MySQL,
p:php

    C/S架构，Client/Server，客户机/服务器；
    B/S架构，Browser/Server，浏览器/服务器；

目的:为了能让我们在浏览器中输入域名的时候,也能访问我们服务器的资源

步骤:
    0.配置apache服务器根目录；
        目的:浏览器中输入 127.0.0.1 访问的不再是wamp安装目录下C:\wamp\www 这个里面的资源,让其访问我们自己的资源；

        更改\apache\Apache2.2.21\conf 目录下面的 httpd.conf 文件,
            改其中两个地方,一个是 DocumentRoot的路径 , 另外一个是 <Directory/> 路径；

    1.开启虚拟主机辅配置，在httpd.conf 中找到 Virtual hosts,去掉第二句前面的#号注释，开启httpd-vhosts虚拟主机模块；

    2.配置虚拟主机，打开conf/extra/httpd-vhosts.conf ；
        分别修改以下三项:
            DocumentRoot "在浏览器中输入域名之后访问的文件夹路径"；
            ServerName "自己配置的域名"；
            ServerAlias "www.自己配置的域名.com"；
        其它项不需要修改

    3.修改本机DNS（hosts）文件
        打开C:\Windows\System32\drivers\etc\hosts(目录是固定的)进行修改

    tips：配置文件修改之后必须要重启wamp服务器，配置才能够生效，否则将会出错；

2、http通信

URL:Uniform Resource Locator;
HTTP:Hypertext Transfer Protocol;
AJAX:Asynchronous JavaScript And XML;

URL:
    （1）、基本概念:
            URL 的全称是 Uniform Resource Locator（统一资源定位符）；
            通过 1 个 URL，能找到互联网上唯一的 1 个资源；
            URL就是资源的地址、位置，互联网上的每个资源都有一个唯一的 URL；

    （2）、URL格式:
            基本格式：协议头://主机地址/路径
                协议头：不同的协议，表示不同的资源查找及传输方式；
                主机地址：存放资源主机的 IP 地址（域名）；
                路径：资源在主机中的具体位置；

HTTP：
    （1）、作用：规定了客户端与服务器数据的传输格式，一般有XML和JSON；
        让客户端和服务器能有效地进行数据沟通；
        HTTP 协议是网络编程使用最为广泛的协议；

        · https协议，对页面进行了加密，相比http协议加载速度更慢；

            旧版本：http 0.9、1.0；
            现通用：http 1.1；

    （2）、HTTP协议特点
           简单快速：因为HTTP协议简单，所以 HTTP 服务器的程序规模小，通信速度快；
        灵活：HTTP 允许传输任意类型的数据；

        HTTP 0.9 和 1.0 使用非持续连接
            限制每次连接只处理一个请求，服务器对客户端的请求做出响应后，立刻断开连接，这种方式可以节省传输时间；

        HTTP 1.1 使用持续连接
            不必为每个web对象创建一个新的连接，一个连接可以传送多个对象；

        tips:持续连接并非连接上就不会断开，http1.1在客户端与服务器连接的时候，有一个时间戳，
        当计时结束的时候，客户端仍然没有新的操作发送到服务器，连接就会自动断开；

    tips：一个url代表了互联网上唯一的一个资源，域名解析控制在政府手中；

Request

请求（Request）：客户端向服务器索取数据的一种行为；
HTTP协议规定：一个完整的由客户端发给服务器的 HTTP 请求 需要包含以下内容：

     （1）、请求行：指定请求方法（GET/POST）、请求资源路径，以及 HTTP协议版本，/表示访问根目录；
                eg：
                   GET / HTTP/1.1

            tips:也称为请求报文行。

     （2）、请求头：对客户端的环境描述、客户端请求的主机地址等信息
                请求头中至少包含，客户端要访问的服务器主机地址；
                           eg:
                            Host: m.baidu.com

                请求头中还可以包含，客户端的类型，客户端的软件环境
                        eg:
                            User-Agent: iPhone AppleWebKit

                客户端所能接收的数据类型：    Accept: text/html, */*
                客户端的语言环境：        Accept-Language: zh-cn
                客户端支持的数据压缩格式：    Accept-Encoding: gzip
                访问结束后，是否断开连接：    Connection: Close

    （3）、请求体(可选)：客户端发给服务器的具体数据，例如要上传的文件数据

        请求格式
            每一项请求信息末尾使用 \r\n
            最后一个请求项末尾使用 \r\n\r\n 表示请求结束

    tips：get方式请求体为空，post方法请求体不为空；

Response

响应（Response）：服务器对客户端的请求做出的反应，一般指返回数据给客户端

HTTP协议规定：一个完整的 HTTP 响应包含以下内容：
    （1）、状态行：包含了HTTP协议版本、状态码、状态英文名称
                eg：请求成功
                    HTTP/1.1 200 OK

                eg：内容无变化
                    HTTP/1.1 304 Not Modified
            tips:一般的，响应状态行的作用在于调试；

    （2）、响应头：包含了对服务器的描述、对返回数据的描述
            服务器的类型：
                Server: Apache/2.4.10 (Unix) PHP/5.5.20
            返回数据的类型
                Content-Type: text/html
            返回数据的长度
                Content-Length: 660
            响应时间
                Date: Thu, 04 Jun 2015 19:31:50 GMT

            ETag(HASH值，用于检测本地资源和服务器资源是否一致)
                ETag: "294-4e1862f57e7c0"

    （3）、实体内容：服务器返回给客户端的具体二进制数据；

4、XMLHttpRequest（170205）

XMLHttpRequest异步请求对象；
无论是前端、iOS、Android，只要是发送网络请求，基本都采用异步方式，因为同步的效率太低；

（1）、使用原生方式发送Ajax请求
    1、创建XHR对象
        var xhr = new XMLHttpRequest();
            tips：该对象是一个浏览器对象，用于局部刷新完成；

    2、发送请求
        请求行    xhr.open("get/post" , "url");
        请求头    xhr.setRequestHeader("Content-Type" , "application/x-www-form-urlencoded");
        请求体    xhr.send(data);

        tips：
            请求行中如果方式为get，
                xhr.open("get",url + data)，将需要发送的数据写在url的后缀上    ，url？name=zhangsan&pwd=123；
                xhr.setRequestHeader()可以不用设置，可以不用设置请求头；
                xhr.send(null)；

            请求行中如果方式为post，
                xhr.open("post","url")；
                xhr.setRequestHeader()需要设置；
                xhr.send(data)，data的值为一个对象；
    3、处理服务器的响应
        xhr.onreadystatechange = function(){
            //当服务器返回了数据的时候触发该事件；
        };
-------------------
        tips：返回的数据都存在了xhr对象中；
            xhr.status        服务器返回的状态码，200表示成功响应；
            xhr.readyState    异步对象状态码，4表示数据已经解析完成并可以使用；

        tips：服务器返回的状态码:
                1xx:消息
                2xx:成功
                3xx:重定向
                4xx:请求错误
                5xx:服务器错误
        tips：异步对象的状态码
                0:异步对象已经创建
                1:异步对象初始化完成,发送请求
                2:接收服务器返回的原始数据
                3:数据正在解析,解析需要时间
                4:数据解析完成,数据可以使用了

        tips:get方式主要用于获取数据，十分不安全，但是get具有缓存，效率快；
            post方式主要用于给服务器传递数据，相对安全，post没有缓存，相率慢；

            get方式将上传的数据存放在URL中，因此数据有长度限制；
            post将上传的数据放在请求体中，理论上没有限制，取决于服务器；

5、xml或json返回数据（170207）

服务器返回的数据通常有两种格式，json或xml；

    XML:Extensible Markup Language
    JSON:JavaScript Object Notation

XML常用于设置文件和游戏score的保存；
JSON 是网络上用来传输数据使用最广泛的数据格式，没有之一；

（1）、xml的解析需要前后台配合:
    1.后台在返回的时候,在响应头中设置Content-Type的值为 application/xml；
    2.前台异步对象在接收后台数据时,记得按照xml的方式来接收,xhr.responseXML,并且它返回的是一个object对象,内容是#document；
        eg:
            <?php
                header("Content-Type:text/xml;charset=UTF-8");
                echo file_get_contents('text_147_XML_Build.xml');
            ?>

（2）、js中解析/操作JSON：
    JSON.parse(json字符串); 将一个json格式的字符串解析成js对象；
    JSON.stringify(js对象); 将一个js对象转成一个json格式的字符串；
        eg:
            var result = JSON.parse(xhr.responseText);
            var resultStr = JSON.stringify(result);

（3）、JSON数据用xhr.responseText，
    XML数据用xhr.responseXML；

（4）、XML
    <?xml version="1.0" encoding="UTF-8" ?>
    下面写xml的自定义标签，标签里面存放数据；
    xml里面的标签可以自定义，但是尽量使用标签存储数据，而不用标签属性存储数据。

        eg:
            xhr.onreadystatechange= function () {
                if(xhr.status==200 && xhr.readyState==4){
                    var result=null;
                    var rType=xhr.getResponseHeader("Content-Type");
                    if(rType.indexOf("xml")){
                        result=xhr.responseXML;
                    }else if(rType.indexOf("json")){
                        result=JSON.parse(xhr.responseText);
                    }else{
                        result=xhr.responseText;
                    }
                }
                //在这里获取到的result，是一个XML文本对象
                callback(result);
            }

6、ajax封装

function  ajax(obj) {    //异步请求封装
    var url = obj.url;
    var method = obj.method.toLowerCase();

    var data = obj.data; 
        //现在就是js对象了 {name:"zhangsan","pwd":123}
        //把上面的data，js对象拼接成键值对的字符串

    var success = obj.success;

    var finalData = pinjieData(data);
        //finalData最终的效果 name=zhangsan&pwd=123

//1.创建xhr对象
    var xhr = new XMLHttpRequest();

    if (method=='get'){
        url = url + "?"+finalData;
        finalData = null;
    }

//2.设置请求行
    xhr.open(method,url);

// 如果是post请求,要设置请求头
    if (method=='post'){
        xhr.setRequestHeader("Content-Type","application/x-www-form-urlencoded")
    }

//3.发送
    xhr.send(finalData);

//4.监听服务器返回的数据
    xhr.onreadystatechange = function () {
        if (xhr.status==200 && xhr.readyState==4){
            var result = null;

        //获取返回的数据类型
            var rType = xhr.getResponseHeader("Content-Type");

            if (rType.indexOf('xml')!=-1){
                result = xhr.responseXML;

            }else if(rType.indexOf('json')!=-1){
                // JSON.parse 的意思是 将 json格式的字符串
                //比如 [{ "src":"./images/nav0.png","content":"商品分类1"}]
                //转成js对象

                result = JSON.parse(xhr.responseText);

            }else{
                //当成普通的字符串去处理
                result = xhr.responseText;
            }
            //将这里解析好的数据交给页面去渲染
            success(result);
        }
    }
}

7、jQuery的$.ajax()方法（170208）

eg:
    $.ajax({
        url:"",
        type:"get/post",
        data:"{}",
        success:function(){

        },
        error:function(){

        },
    })

    url :接口地址
    type :请求方式(get/post)
    timeout : 要求为Number类型的参数，设置请求超时时间（毫秒）
    dataType: 应该是客户端传递给服务器一个值，告诉服务器如何进行处理:
    data: 发送请求数据

    beforeSend: 要求为Function类型的参数，发送请求前可以修改XMLHttpRequest对象的函数，
    例如添加自定义HTTP头。在beforeSend中如果返回false可以取消本次ajax请求.
    success: function(){}，成功响应后调用
    error: function(){}，错误响应时调用
    complete: function(){}，响应完成时调用（包括成功和失败）

9、使用script实现跨域（170210）

（1）、概念:
    同源：同源策略是浏览器的一种安全策略，所谓同源是指，域名，协议，端口完全相同 
    跨域：不同源则该行为是跨域；

tips：在Ajax请求的时候，不允许跨域请求；

src属性和href属性有天生跨域的特性，<img> <script> <link> <a>；
script的src属性可以访问外链，一般通过该特性实现跨域。

（2）、原理:
    1.本质是使用script中src的天然跨域特性
    2.客户端,通过GET请求,url传递一个客户端的方法到服务器,
    服务器再返回这个方法的执行代码到客户端,最终在客户端进行调用执行；

    3.方法的执行本质是在客户端执行的
    eg:
        <script src="http://www.link.com?callback="function_Name"&date=date "></script>

（3）、采用jsonp实现跨域，需要和后台定义返回的数据格式，jsonp只能用get方式发送；
        返回的数据格式为
                eg:    callback({
                        key:value,
                        key:value
                    })


tips：为什么跨域要在script的src中解决，而不是link的href中，那是因为script比起link，它能执行方法

tips：在ajax中,如果使用了dataType:jsonp,那么当前请求就和XMLHTTPRequest没有任何关系了，jquery自动采用script的src属性实现了跨域；
